## Phase 3 â€” Accounts & entitlements (the paid-tier backbone, ~1 week)

**Goal**: sign in to the desktop app with the site's Clerk account; paid

No Paddle, no separate license keys â€” the billing system already exists.

### Device-token flow (OAuth-style, no embedded webview)

1. App: "Sign in" â†’ opens
   `https://accessibility.build/desktop/connect?device=MacBook%20Pro` in the
   default browser (Clerk-protected page).
2. Page (signed-in user) generates a one-time code, stores a **hashed**
   device token in a new `desktop_devices` table
   (`id, user_id, token_hash, device_name, created_at, last_seen_at,
   revoked_at`), then redirects to
   `accessibility-build://auth?token=<token>`.
3. App registers the `accessibility-build://` scheme
   (`tauri-plugin-deep-link` + `CFBundleURLTypes` in Info.plist), receives
   the token, stores it in the macOS Keychain (via `security` /
   tauri-plugin-keychain â€” never plaintext on disk).
4. All API calls send `Authorization: Bearer <token>`.

Account page on the site lists connected devices with revoke buttons
(`revoked_at` â€” instant kill switch).

### API surface (new, all under `app/api/desktop/`)

| Route | Purpose |
|---|---|
| `POST /api/desktop/connect` | exchange one-time code â†’ device token |
| `GET  /api/desktop/entitlements` | `{ plan, credits, features: { urlScans, aiAltText, cloudReports } }` â€” derived from existing `users`/credits |
| `POST /api/desktop/ping` | opt-in anonymous `{version, tools_used}` telemetry |
| `GET  /downloads/desktop/latest.json` | updater manifest (static, no-cache) |

App-side: settings card shows account, credits, sign-out; entitlements
cached with a TTL so the app works offline on last-known state.

## Security notes

- Device tokens: random 256-bit, stored hashed (sha256), transmitted only
  over TLS, revocable, `last_seen_at` updated on use.
- Keychain storage on the client; never in localStorage or the config dir.
- Rate-limit `/api/desktop/*` per token; entitlements response is cacheable
  but never authoritative for credit *deduction* (server debits).
- Telemetry strictly opt-in, no screen content ever.
